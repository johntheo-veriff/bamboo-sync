# Events go in the User's primary Google Calendar

Calendar Events are created in the User's primary Google Calendar rather than a dedicated "BambooHR" calendar. The core use case is scheduling — colleagues need to see a User as busy during time-off when booking meetings. A dedicated calendar would require every colleague to have it visible, which can't be enforced. Using the primary calendar means availability is reflected correctly by default.

## Considered options

- **Dedicated calendar** — cleaner separation, easy to delete all events at once, but invisible to others unless explicitly shared/subscribed. Doesn't solve the scheduling problem.
- **Primary calendar** — chosen. Events appear in scheduling views automatically. Colour-coding distinguishes bamboo-sync events from personal ones.
