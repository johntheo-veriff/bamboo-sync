# Bamboo Sync

A web app that syncs a user's BambooHR time-off and company holidays into their personal Google Calendar. One-way, ongoing, self-serve.

## Language

**User**:
A company employee who connects their BambooHR account and Google Calendar to bamboo-sync.
_Avoid_: person, member, account

**BambooHR Subdomain**:
The company-specific subdomain of BambooHR (e.g. `acme` in `acme.bamboohr.com`). Entered by the User during onboarding to identify which BambooHR account to connect to.
_Avoid_: domain, company name, URL

**BambooHR API Key**:
A personal API key generated by a User from their BambooHR profile settings. Scoped to that user's own data. Used by bamboo-sync to authenticate with BambooHR on behalf of the User.
_Avoid_: token, credential, password

**Time-off Entry**:
A single approved leave record in BambooHR (e.g. PTO, sick day). Synced to the User's Google Calendar as an event.
_Avoid_: leave, absence, vacation request

**Holiday**:
A company-wide non-working day defined in BambooHR by HR. Synced to the User's Google Calendar as an event.
_Avoid_: public holiday, bank holiday, day off

**Connection**:
The active link between a User's BambooHR credentials (Subdomain + API Key) and their Google Calendar. Established once during onboarding; persists in the background with no bamboo-sync login required. Disconnecting removes the Connection and deletes all Calendar Events bamboo-sync created.
_Avoid_: integration, setup, account, session

**Calendar Event**:
An event created by bamboo-sync in the User's primary Google Calendar, representing either a Time-off Entry or a Holiday. Time-off Entries and Holidays use different colours to distinguish them at a glance. If a colour is unavailable, both fall back to the same colour.
_Avoid_: entry, appointment, block

**Sync**:
The ongoing background process that reads Time-off Entries and Holidays from BambooHR and mirrors them in the User's Google Calendar — creating, updating, and deleting events to match BambooHR's current state. Only covers today and future dates; past entries are not synced. Runs once per day.
_Avoid_: import, export, transfer, job

**Management Page**:
A minimal UI shown to the User after onboarding. Displays the current Connection status, the time of the next scheduled Sync, and options to update the BambooHR API Key or disconnect. Shows an error state if the last Sync failed (e.g. invalid API Key).
_Avoid_: dashboard, settings, profile, admin panel

## Example dialogue

> **Dev**: "So when a User sets up bamboo-sync, do they need to create an account?"
> **Domain expert**: "No — there's no bamboo-sync account. They establish a Connection by entering their BambooHR Subdomain and API Key, then authorising Google. That's it."
>
> **Dev**: "What gets created in Google Calendar?"
> **Domain expert**: "Calendar Events — one per Time-off Entry or Holiday. They land in the User's primary Google Calendar, colour-coded so they stand out."
>
> **Dev**: "What if someone's holiday gets cancelled in BambooHR?"
> **Domain expert**: "The next Sync deletes the Calendar Event. BambooHR is always the source of truth."
>
> **Dev**: "How does the User know the Sync is working?"
> **Domain expert**: "They open the Management Page. It shows the Connection status and when the next Sync will run. If something broke, they see the error there."
